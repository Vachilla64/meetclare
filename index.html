How it works

Just really good at SQL.

Clare runs on Coral, an open-source engine that turns live APIs into SQL tables. No syncing, no pipelines, no drama. Your tools show up as database tables and Clare reads them right now.

Under the hood

Powered by Coral

Clare is built on Coral, an open-source SQL runtime that turns live APIs into database tables. GitHub issues, calendar events, Slack messages — all queryable with plain SQL, all in real time.

-- Clare hears: "What does my day look like?"

SELECT summary, status
  FROM google_calendar.events
  LIMIT 5;

SELECT number, title, state
  FROM github.issues
  WHERE owner='withcoral'
    AND state='open'
  LIMIT 5;

-- No database setup. No pipelines. Just answers.
✦

Everything in one query

GitHub, Calendar, and Slack all show up as SQL tables. One question, three live sources, no glue code required.

✦

Auto-discovered schema

Coral maps out exactly what tables and columns are ready to query. Clare always knows what she's working with.

✦

Smart caching

Repeated queries hit Coral's cache instead of hammering your APIs. Fast replies without burning through rate limits.

✦

Hand-rolled Google Calendar connector

Clare ships with a custom Google Calendar source spec built for this project. Not in Coral's standard library. Written from scratch.

✦

Your data stays yours

Queries run in your own environment. Clare does not send your credentials or data anywhere she shouldn't.
